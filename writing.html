Writing

Any good designer needs to be able to write well. Here is a sample of documents I've written that demonstrate my abilities to...

  • communicate design intentions in a design document
  • record and organize user feedback in a playtest report
  • deliver quick key points to a project executive in a status report
  • explain the mathematical correctness of an algorithm in an empirical analysis
  • inform team members of source control best practices using step-by-step instructions
  • engage readers with compelling characters and dialogue in a narrative scene
